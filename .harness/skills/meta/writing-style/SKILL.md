# Writing Style Skill

Provides full writing style reference for generating authentic, on-voice documentation.

## Quick Reference

### Templates

**Feature Description:**
```
[Feature Name]: [What it does]. [Benefit or use case].

[Optional caveats]
• Point 1
• Point 2
```

**Requirement Entry:**
```
**[Category Name]**
1. **[Specific Requirement]:** [Description with action verb]
   1. **[Sub-requirement]:** [Details]
```

**Problem/Context Setup:**
Context → Challenge → Hint at solution

### Preferred Phrases

| Use | Instead of |
|-----|------------|
| "Ability to..." | "The ability to..." |
| "Users can..." | "Users are able to..." |
| "If enabled..." | "When this is turned on..." |
| "This potentially boosts..." | "This will improve..." |
| "We are adding..." | "We will be adding..." |
| "Integration with..." | "Integrates with..." |

### Common Openings
- "We are adding..." / "We are introducing..."
- "This feature enables..." / "This feature leverages..."
- "Users can..." / "Users have the capability to..."
- "The app now..." / "The system provides..."
